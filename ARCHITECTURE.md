## Key Design Patterns

1. **Separation of Concerns**
   - UI Components (React)
   - Business Logic (utilities)
   - Data Storage (Tauri/Rust)

2. **Composition**
   - Small, reusable components
   - MOAModelConfig â†’ MOAScenarioBuilder â†’ ScenarioManager

3. **Provider Pattern**
   - aiClient abstracts provider details
   - Easy to add new AI providers

4. **Strategy Pattern**
   - Sequential vs Parallel execution
   - Configurable at runtime

5. **State Management**
   - Centralized Zustand store
   - Component-level state where appropriate

6. **Security**
   - Input validation
   - Filename sanitization
   - Error handling
